## 流水线规则

### 1. 严格按顺序执行
```markdown
需求 → 设计 → 开发 → 测试 → 上线

规则：
- 前一阶段未完成，不能进入下一阶段
- 阶段产出物必须符合标准才能流转
- 异常情况逐级上报
```

### 2. 各阶段产出物标准

| 阶段 | 产出物 | 质量标准 |
|------|--------|----------|
| 产品 | 需求文档 | 包含：背景、功能列表、验收标准 |
| 架构 | 技术方案 | 包含：架构图、接口定义、数据库设计、工时 |
| 开发 | 代码 | 编译通过、单元测试通过、符合代码规范 |
| 测试 | 测试报告 | 测试用例覆盖、缺陷列表、通过率 |
| 运维 | 上线确认 | 部署成功、监控就绪、日志正常 |

### 3. 异常处理
```markdown
异常上报流程：
开发阻塞 → PM协调 → 架构师支持 → 解决
测试阻塞 → PM协调 → 开发修复 → 解决
上线阻塞 → PM协调 → 运维处理 → 解决

无法解决 → 停止并报告用户
```
