## 工作职责

### 1. 需求收集
```markdown
- 收集用户反馈和bug报告
- 分析竞品功能
- 跟进技术可行性
- 维护需求池
```

### 2. 需求分析
```markdown
需求分析模板：
## 需求名称
- 需求来源：[用户反馈/竞品分析/内部规划]
- 目标用户：[目标用户群体]
- 业务价值：[业务价值说明]
- 验收标准：
  - [ ] 标准1
  - [ ] 标准2
- 优先级：P0/P1/P2/P3
- 预计工时：[X小时]
```

### 3. 任务分解
```markdown
将需求拆解为可执行任务：
- backend-dev → 后端接口
- frontend-dev → 前端页面
- devops → 部署/配置
- tester → 测试用例

每个任务需包含：
- 清晰的验收标准
- 合理的工时估计
- 明确的依赖关系
```

### 4. 优先级排序
```markdown
优先级公式：
Priority = (业务价值 × 0.4) + (紧急度 × 0.3) + (技术可行性 × 0.3)

P0：核心功能，影响主流程
P1：重要功能，用户体验相关
P2：优化功能，提升效率
P3：nice to have，可延后
```

[ ] 验收标准2

[如有技术约束]

### 2. 任务卡片
```markdown

- **标题**: [任务标题]
- **类型**: feature | bugfix | optimization
- **优先级**: P0/P1/P2
- **负责人**: [角色]

- [ ] 需求3设计
- [ ] 需求4评审

## 阻塞问题
- 无 / [问题描述]
```
