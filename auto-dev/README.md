## 流水线规则

### 阶段顺序
```
需求 → 设计 → 开发 → 测试 → 上线
  │      │      │      │      │
  ▼      ▼      ▼      ▼      ▼
产品   架构   开发   测试   运维
```

### 规则
1. **严格按顺序执行** - 前一阶段未完成，不能进入下一阶段
2. **产出物达标才能流转** - 每个阶段有质量标准
3. **异常逐级上报** - 阻塞问题通过PM协调解决

| Agent | 角色 | 核心职责 | 输入 | 输出 |
|-------|------|----------|------|------|
| **Project Manager** | 项目经理 | 用户交互、进度协调 | 用户需求 | 任务分发 |
| **Product Manager** | 产品经理 | 需求分析 | 用户需求 | 需求文档 |
| **Architect** | 架构师 | 技术设计 | 需求文档 | 技术方案 |
| **Backend-Dev** | 后端开发 | Java/Spring | 技术方案 | 后端代码 |
| **Frontend-Dev** | 前端开发 | Vue3/TS | 技术方案 | 前端代码 |
| **UI-Designer** | UI设计师 | 界面设计 | 设计要求 | 设计规范 |
| **Tester** | 测试工程师 | 测试验证 | 构建产物 | 测试报告 |
| **DevOps** | DevOps | 构建部署 | 代码 | 构建产物 |
| **Operation** | 运维 | 运维上线 | 测试报告 | 上线完成 |
| **Orchestrator** | 协调者 | 流水线协调 | - | - |

## 流水线任务示例

用户说："需要增加一个导出Excel功能"

```
阶段1：产品分析（Product Manager）
- 输出：需求文档
  ## 导出Excel功能需求
  - 功能：导出用户列表为Excel
  - 字段：ID、姓名、手机、邮箱
  - 验收：下载成功、格式正确、中文正常

阶段2：架构设计（Architect）
- 输出：技术方案
  ## 技术方案
  - 使用Apache POI导出Excel
  - 接口：GET /user/export
  - 数据库：无需修改

阶段3：开发实现（Backend-Dev + Frontend-Dev）
- 输出：代码
  - UserExportController.java
  - export.vue

阶段4：测试验证（Tester）
- 输出：测试报告
  - 功能测试通过
  - 边界测试通过

阶段5：上线发布（Operation）
- 输出：上线完成
  - 部署成功
  - 监控就绪
```

## 与Orchestrator通信

```bash
# 你只需要告诉项目经理（Project Manager）

"需要XXX功能"         → PM接收需求
"这个功能有Bug"        → PM记录Bug
"进度怎么样了"         → PM汇报进度
"优先做XXX"           → PM调整优先级
"暂停一下"            → PM暂停流水线
```
