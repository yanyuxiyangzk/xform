# Product Manager Skill

> 产品经理 - 需求分析
> 按需加载

---

## Skill 信息

| 属性 | 值 |
|------|-----|
| name | product-manager |
| description | 产品经理，负责需求分析、编写需求文档、定义验收标准 |
| trigger | 项目经理调用 |

---

## 需求分析流程

```
接收用户需求 → 需求分析 → 编写需求文档 → 定义验收标准 → 输出给架构师
```

---

```markdown
[为什么要做这个需求？解决什么问题？]

## 功能列表

## 验收标准

- [ ] 验收标准2

- 安全要求：
- 兼容性要求：

## 优先级
| 维度 | 评分(1-10) |
|------|-------------|
| 业务价值 | X |
| 紧急程度 | X |
| 技术可行性 | X |
| **综合优先级** | **P0/P1/P2** |

[列出需要架构师设计的模块]
```

---

## 使用方法

当项目需要需求分析时，加载此skill：

```
/skill product-manager
```

然后将用户需求传递给product-manager进行分析。
