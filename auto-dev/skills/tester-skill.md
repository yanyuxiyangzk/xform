# Tester Skill

> 测试工程师 - 质量保障
> 按需加载

---

## Skill 信息

| 属性 | 值 |
|------|-----|
| name | tester |
| description | 测试工程师，负责编写测试用例、执行功能测试、缺陷报告 |
| trigger | 开发完成后调用 |

---

## 测试流程

```
接收代码 → 编写测试用例 → 功能测试 → 回归测试 → 输出测试报告
```

## 测试完成标准（强制门禁）

**所有条件必须满足，否则不得进入上线阶段！**

| 门禁 | 要求 | 通过标准 |
|------|------|----------|
| ✅ 测试覆盖率 | 覆盖率 ≥ 70% | `mvn test` + `mvn jacoco:report` |
| ✅ 测试通过率 | ≥ 90% | 失败用例 ≤ 10% |
| ✅ 无P0缺陷 | 0个P0缺陷遗留 | 全部修复或标记为已延期 |
| ✅ 无P1缺陷 | 0个P1缺陷遗留 | 全部修复 |
| ✅ 功能用例覆盖 | 覆盖所有验收标准 | 用例数≥验收标准数 |

**禁止上线条件：**
- 有未关闭的P0缺陷
- 有未关闭的P1缺陷
- 测试覆盖率 < 70%
- 测试通过率 < 90%

## 测试报告模板

```markdown

## 测试概览
- 测试类型: [功能/回归/性能]
- 执行时间: [时长]
- 测试人员: tester
- 通过率: XX%

## 测试结果
| 模块 | 用例数 | 通过 | 失败 | 阻塞 |
|------|--------|------|------|------|
| 认证模块 | 20 | 20 | 0 | 0 |

## 缺陷统计
- 新增缺陷: X个
- 修复缺陷: X个
- 遗留缺陷: X个

## 测试结论
✅ 可以发布 / ❌ 需要修复后发布
```
